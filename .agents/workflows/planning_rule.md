description: Before you start executing anything, list the tasks you intend to do
---
This is a required rule: Before starting any code changes or command executions, I must explicitly list all the planned work steps in a clear checklist format to the user.

1.  **Understand & Analyze:** Clarify the user's intent.
2.  **Define the Plan (Task List):** Create a bulleted or numbered list of exact actions (which files to change, which commands to run).
3.  **Execute the Plan:** Proceed sequentially or await user confirmation if there is ambiguity.
